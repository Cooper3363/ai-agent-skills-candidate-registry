## 两级验收结论定义

### 1. 候选试跑通过

候选试跑通过只代表该对象可以进入内部/受控试跑，不代表客户正式可调用，不代表进入稳交付库。

最低门槛：

- 有完整 `CANDIDATE.md` / `candidate.yaml` 或 `SKILL.md` / `skill.yaml`。
- 能被 DeepAgents / 通用 Agent Skill 底座调用，且调用方式写清。
- 能使用 OpenAI-compatible 中转站模型通道，且不得在包内写入真实 key。
- 有稳定 candidate/skill/agent ID。
- 有清晰 intent / role / business task。
- 有稳定输入字段、输出字段和最小中文调用样例。
- 明确 trial mode：mock、dry-run、draft_l3、agent_role、media_generation_plan 等。
- 明确权限边界、禁止动作、人工复核触发、失败模式、日志/审计建议。
- 明确 BYOK、OAuth、API、Provider 费用与权限是否需要。
- 若涉及外部 API 或真实执行，必须默认 dry-run，且真实执行开关默认关闭。
- 能服务中小微六部门高频业务，不只是技术演示或泛工具玩具。

可用结论：

- `candidate_trial_accepted`
- `candidate_trial_needs_fix`
- `blocked`
- `component_only`

### 2. 可建议稳交付提升

可建议稳交付提升只代表平台验收窗口建议进入稳交付评审，不等于自动改变客户正式可调用数量。客户正式可调用数量必须在平台验收明确批准并完成入库动作后才可变更。

最低门槛：

- L1 来源、许可证、商用边界清楚。
- L2 中文业务用例通过；若只是 smoke passed，不能写成正式 L2 通过。
- L3 标准封装完整，含 `SKILL.md` / `skill.yaml` 或等价 Agent Skill 封装。
- 平台最小中文调用样例可执行，输入输出稳定。
- DeepAgents / 通用 Agent Skill 适配路径清楚。
- OpenAI-compatible 中转站模型通道可配置、可审计，不内置真实 key。
- 权限、费用、API、OAuth/BYOK、日志、失败处理、重试、限额、人工复核触发清楚。
- 对外部 API、真实图片/视频生成、数字人、发送、写入、投放、支付、账号、CRM 等动作有显式开关和审计记录。
- 默认不隐式执行高风险动作；真实执行必须由平台策略、用户授权和运行时权限共同控制。
- 有清楚的回滚/失败降级策略。

可用结论：

- `stable_promotion_recommended`
- `stable_promotion_needs_fix`
- `stable_promotion_needs_test`
- `stable_promotion_needs_license_review`
- `stable_promotion_blocked`

## 核心验收维度

| 维度 | 候选试跑要求 | 稳交付提升要求 |
| --- | --- | --- |
| DeepAgents / 通用 Agent Skill 调用 | 写清 adapter 或调用方式即可试跑 | 需有可执行最小调用样例和运行时交接说明 |
| OpenAI-compatible 中转站模型通道 | 写清模型通道配置，不写真实 key | 需可配置、可审计、可限额、可失败降级 |
| 输入输出 | 稳定 schema 与中文样例 | 多场景稳定输出，字段语义可消费 |
| BYOK/OAuth/API | 说明是否需要 | 明确 provider、权限、费用、限额、日志、失败处理 |
| dry-run 与真实执行 | 默认 dry-run，真实执行关闭 | 真实执行必须显式授权、可审计、可回滚或可停止 |
| 人工复核 | 触发条件明确 | 高风险场景必须进入人工复核或阻断 |
| 业务价值 | 对应六部门高频任务 | 有明确中小微业务流程落点和重复使用价值 |
| 风险控制 | 禁止动作写清 | 权限、内容、费用、合规、审计均可控 |

## 图片/视频/数字人额外验收重点

### 候选试跑要求

- 明确 provider / API / 模型来源。
- 明确是否需要 BYOK 或平台统一 key。
- 明确是否可通过 OpenAI-compatible 中转站模型路由。
- 明确费用、限额、失败重试、超时和降级策略。
- 明确生成内容不得自动发布、不得自动投放、不得自动商用承诺。
- 明确版权、肖像、商标、品牌素材、第三方素材授权风险。
- 明确内容审核机制：违法违规、成人、暴力、仇恨、政治敏感、医疗金融误导、未成年人、名人肖像等。
- 有最小真实生成样例计划；若未执行真实生成，必须标注为 plan 或 dry-run。

### 稳交付提升要求

- provider/API 条款、商用授权、数据使用和内容所有权边界已复核。
- 版权、肖像、商标、素材授权风险有明确提示和人工复核条件。
- 内容审核链路明确，失败时阻断或转人工。
- 费用、限额、重试、超时、失败降级、日志审计可由平台控制。
- BYOK / OAuth / 平台统一 key 的责任边界清楚。
- 最小真实生成样例通过，且输出不被误写成商用授权证明。
- 不承诺生成内容一定可商用，不替代法务、品牌、平台政策审核。

## 高风险动作硬边界

以下动作不得在候选试跑中隐式执行，稳交付也必须通过平台权限、用户授权和审计控制：

- 自动发送邮件、短信、私信、外呼。
- 自动写 CRM、客服工单、财务系统、HR 系统、广告账户、电商后台。
- 自动退款、赔偿、付款、下单、改库存、改价格、改合同。
- 自动发布营销内容、广告投放、视频/图片上线。
- 自动处理账号安全、身份验证、密码、验证码、恢复流程。
- 输出法律、税务、审计、融资、投资、医疗、招聘录用/淘汰等确定性高风险结论。
