# 许可证检查表

## 首批可进入 L1 的许可证

- MIT
- Apache-2.0
- BSD-2-Clause
- BSD-3-Clause

## 首批延期的情况

- GitHub API 返回 `NOASSERTION`。
- 使用 CC-BY、GPL、AGPL、SSPL、BUSL、自定义商业限制许可证。
- 仓库已归档。
- README 声称开源，但未提供明确 LICENSE。
- 代码许可证宽松，但模型权重、数据集、品牌素材或云服务条款另有限制。

## 核验步骤

1. 打开官方仓库主页，确认未归档。
2. 查看 GitHub license SPDX 识别结果。
3. 打开 `LICENSE` 文件，确认与 SPDX 一致。
4. 检查 README 是否存在额外商用限制。
5. 检查核心依赖、模型权重、数据集和第三方 API 条款。
6. 在 `skill.yaml` 或入库卡片中记录核验日期与风险备注。

## 商用备注

本检查表用于产品筛选，不构成法律意见。进入正式商业分发、托管运行或面向客户销售前，应由法务复核源项目许可证和依赖许可证。
